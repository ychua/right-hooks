# Engineering Review: feat/stats

Generated by /plan-eng-review on 2026-03-23
Branch: feat/stats | Base: main

## Summary

7 issues found (1 Critical, 2 High, 1 Medium, 3 Low). The event recording system
is well-designed but has several bugs that need fixing before merge.

## Critical (P0)

### 1. Write failure in `rh_record_event` can abort hooks

**File:** `hooks/_preamble.sh` line ~112
**Problem:** If the `echo >>` to the JSONL file fails (disk full, permissions), the
hook aborts instead of continuing. This violates the fail-open contract — a stats
recording failure should never block enforcement.
**Fix:** Add `2>/dev/null || true` to the echo line.

## High (P1)

### 2. Simple hooks not wired for auto-recording

**Files:** `post-edit-check.sh`, `pre-pr-create.sh`, `pre-push-master.sh`, `subagent-stop-check.sh`
**Problem:** These hooks don't pass the 3rd gate argument to `rh_pass`/`rh_block`,
so stats only capture pre-merge and stop-check events. Other hooks are invisible.
**Fix:** Add gate names as 3rd argument (~10 lines across 4 files).

### 3. `&&`/`||` chain in pre-merge.sh can double-record

**File:** `hooks/pre-merge.sh`
**Problem:** If `rh_record_event` fails after recording "pass", the `||` branch also
fires and records a spurious "block" event. Stats data gets corrupted.
**Fix:** Replace `&&`/`||` chains with explicit if/else (~18 lines across 6 gate checks).

## Medium (P2)

### 4. JSON injection via branch name

**File:** `hooks/_preamble.sh` (rh_record_event)
**Problem:** Branch names containing `"` characters produce malformed JSONL. Low risk
because husky pre-push enforces branch naming, but defense-in-depth says sanitize.
**Fix:** Escape `"` in branch name before interpolation (1 line).

## Low (P3)

### 5. `_RH_BRANCH` subprocess runs unconditionally

Could be lazy-initialized to avoid subprocess on every hook invocation. 1 line fix.

### 6. `mkdir -p` runs on every recording call

Could be guarded with `[ -d ]` check to skip the syscall when dir exists. 1 line fix.
