## Architecture Decisions

### Adapter Interface
```
Runtime (Claude Code / Cursor / Codex / Aider / Windsurf)
     │
     ▼
┌──────────────┐
│  Adapter      │ ← translates runtime format → HOOK-CONTRACT v1
│  (per-runtime)│
└──────┬───────┘
       │ JSON stdin + exit code
       ▼
┌──────────────┐
│  Hooks        │ ← unchanged
└──────────────┘
```

### VCS Abstraction
Preamble helpers (rh_fetch_comments, rh_check_ci, rh_get_pr_body, etc.) abstract `gh` calls. Default implementation uses GitHub. Future implementations for GitLab/Bitbucket swap the VCS backend without touching hook logic.

### API Batching
Fetch all PR comments once per hook invocation, cache in a shell variable, filter locally with jq. Reduces 8+ API calls to ~3-4.
